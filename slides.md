# Why bother?

**high dimensions** allow <span v-mark.green="1">precise and thorough</span> analysis and troubleshooting, we want them!

<v-clicks at="2" every="1" depth="2">

- dimensions are **indexed** to allow querying for analysis, troubleshooting, etc.
- ‚û°Ô∏è high dimension **induces high cardinality**
- ‚û°Ô∏è high cardinality induces **larger indices**
- ‚û°Ô∏è larger indices are **costly to maintain and use**
- costs are
  - **computational costs** (it is long to re-index and **may be** long to request ‚è≥‚è≥)
  - **financial costs** (We pay for *Dynatrace* **compute usage**, including indexing and querying üí∏üí∏)
</v-clicks>

<v-click at="9">

<div mt-10 border-2 border-red-300 text-center p-3>

### We **MUST** be careful with which dimension we declare.
</div>
</v-click>

# **Metrics**: Application-oriented

Sensors for app, OS, database, network traffic, hardware usage, etc

<v-click>

### üèÖ 4 golden signals üèÖ

# **Traces**

*High-level* and *In-depth* analysis for distributed systems

<v-click>

We want to have the **most complete view** and path requests take in our application.  
If the request go through 5 REST APIs and 3 FaaS, we want spans for <span v-mark="{at: 2, color: '#fca5a5', type: 'highlight', iterations: 1, animationDuration: 450}">*(quite)* all underlying components</span>. ü§ì
</v-click>
<v-click at="3">

But, what if we generate a huge amount of traces? Like more than 1 000 per seconds?
</v-click>

::left::

<v-click at="4">

#### **Head Sampling**
</v-click>

<v-clicks at="5">

- When a trace is created, Deterministic Sampling
- üëç Efficient
- üëç Easier to configure
- üëç May be done at any point
- üëé Not possible to ensure that all error traces are sampled

</v-clicks>

::right::

<v-click at="10">

  #### **Tail Sampling**
</v-click>

<v-clicks at="11">

- When a trace is received, done by Observability backend
- üëç Always sample a trace with an error
- üëç Sampling based on overall latency
- üëç Sampling based on specific dimensions values
- üëé More difficult to implement
- üëé May induce more costs (financial and computational)
</v-clicks>
